**Typography:**
- Display / headlines: **Bebas Neue** (Google Fonts) — cinematic, aggressive
- Body / manifesto text: **Syne** (Google Fonts, variable weight) — readable at paragraph scale
- Labels / metadata: **Space Mono** or **DM Mono** — the code-block feel that mirrors brunocis.co's `<code>` labels
- Load via Google Fonts, `font-display: swap`

**ECE design signature:** One subtle nod to electronics throughout — PCB trace lines as section dividers, a sine wave somewhere, or grid lines that look like oscilloscope graph paper. One. Not everywhere.

The `[at]` display trick brunocis.co uses is clever but not enough — the actual mailto still exposes the address in DOM. For Udit Raj, decode from Base64 at runtime:
```javascript
const email = atob('dWRpdHJhamthc2h5YXA0NDJAZ21haWwuY29t'); // encode your real email once, store result
document.querySelectorAll('.email-reveal').forEach(el => {
  el.textContent = email;
```


### 6.3 Anti-Clickjacking
```html
<meta http-equiv="X-Frame-Options" content="DENY">
<meta http-equiv="X-Content-Type-Options" content="nosniff">
```

### 6.4 GitHub API — Rate Limiting
Use Next.js ISR (`revalidate: 3600`). Projects fetched at build time, cached, rebuilt hourly. Users never hit the GitHub API directly. Store `GITHUB_TOKEN` in Vercel env vars for 5000 req/hr headroom.

### 6.5 Environment Variables
Never commit `.env`. Add to `.gitignore` before first commit. Store in Vercel dashboard:
- `GITHUB_TOKEN`
- `WEB3FORMS_ACCESS_KEY` (if contact form added later)

### 6.6 Dependencies
Run `npm audit` before every deploy. Keep GSAP, Lenis, Next.js up to date.

### 6.7 No Personal Data Exposure
- Email: Base64-obfuscated (§6.1)
- Phone: do NOT include on the public site
- Location: city only ("Delhi, IN")
- VIT roll number / reg number: never on site

### 6.8 HTTPS
Deploy to Vercel — HTTPS automatic and enforced. Add to `<head>`:
```html
<meta http-equiv="Strict-Transport-Security" content="max-age=31536000; includeSubDomains">
```

## 7. RESPONSIVE & PERFORMANCE

### Breakpoints
```
Mobile:   < 640px   — single column. Hero manifesto font: clamp(24px, 6vw, 48px)
Tablet:   640–1024px — transition layout
Desktop:  > 1024px  — full layout as designed
Wide:     > 1440px  — max-width 1400px, generous side margins
```

### Mobile Adaptations
- Marquee project titles: reduce to `clamp(40px, 10vw, 80px)` — still dramatic
- Dual-column "good at / suck at": stack to single column below 640px
- Custom cursor: disabled on touch devices
- Logo strip: still works on mobile (CSS animation, no interaction needed)

### Performance
- Hero font (Bebas Neue) preloaded: `<link rel="preload" as="font">`
- `font-display: swap` on all Google Fonts
- `loading="lazy"` on all project visuals
- Inline critical CSS for above-fold content (background color, font-face declaration)
- GSAP/Lenis: `defer` attribute — non-blocking

### Accessibility
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```
All interactive elements: `:focus-visible` styles. Semantic HTML (`<main>`, `<section>`, `<nav>`). Color contrast minimum 4.5:1.
